Отчёт о проверке кодировок и восстановлении текста
====================================================
Дата: 2025-02-21

Масштаб
-------
Проверено .txt файлов: 343 (вся папка Оцифровка).

Тип A (неверная кириллица в UTF-8)
Признак: в файле встречаются символы сербской/македонской кириллицы или типографские знаки вместо русских букв (М, О, Г, г, П, Н, К, А и т.д.).
Обнаружено файлов: 61.
Восстановлено: 61 (таблица замен как для файла Гуровой + контекстные правки).
Список путей: encoding_type_a_files.txt

Тип B (CP1251 / Latin-1 mojibake)
Признак: файл невалиден как UTF-8 и успешно декодируется как CP1251, либо в тексте видны типичные mojibake-последовательности (ãî, ñî, ðî и т.п.).
Обнаружено файлов: 89.
Восстановлено: 89 (чтение байтов, decode('cp1251'), сохранение в UTF-8).
Список путей: encoding_type_b_files.txt

Итог
----
Всего изменено файлов: 61 (Type A) + 89 (Type B) = 150.
Остальные файлы не изменялись (кодировка признана корректной).

Второй проход (довосстановление)
Часть файлов после первой волны оставалась с артефактами: 001_Список текстов, Зиммель «Большие городаЕ», Ганжа «Mobilis in mobili», задание Кноблиха и др. Они были добавлены в список Type A; по ним и по полному списку из detect_encoding.py повторно применён fix_type_a.py (ещё 68 файлов). После повторного detect_encoding.py проблемных .txt не осталось (в списках фигурирует только сам encoding_restoration_report.txt из‑за слов «Type A»/«Type B» в тексте).

Скрипты
-------
detect_encoding.py   Ч сканирование и запись списков Type A / Type B.
fix_type_a.py        Ч восстановление Type A (таблица замен + контекст).
fix_type_b.py        Ч перекодирование Type B (CP1251 → UTF-8).
fix_type_b_undo.py   Ч откат ошибочного Type B (не использовался: Зиммель оказался смесью Type A и UTF-8).
