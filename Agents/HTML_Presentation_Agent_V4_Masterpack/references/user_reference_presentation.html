ОПРЕДЕЛЕНИЕ ДОЛЖНОСТИ

Что делает агент?

Вопросы для определения должности:
1. Какую 'работу' выполняет этот сотрудник?
2. Какие задачи делает ежедневно?
3. Что на выходе? (пост, отчёт, решение)
4. Критерии качества — как понять что хорошо?
Пример: Копирайтер

Работа: Создание текстового контента
Задачи: Посты для соцсетей, кейсы, статьи
Выход: Текст готовый к публикации
Качество: Соответствует tone of voice, цепляет, без воды
Критерии найма:
✅ ЧТО МОЖЕТ: Писать посты, адаптировать стиль, использовать фреймворки
❌ ЧТО НЕ МОЖЕТ: Вести переговоры, принимать стратегические решения
🎯 ЗОНА РОСТА: Понять какие посты заходят, адаптировать хуки

ROI: Экономия 956,400₽/год. Окупаемость курса: 1 день.

Этот подход помогает чётко понять зачем нужен агент и что он даёт.

САМЫЙ ВАЖНЫЙ ПРИНЦИП

Система самообучения

Главное правило:
Если запрос имеет логику → сохранить в контекст

Это отличает живую систему от статичного промпта.

  • 1. Сначала делаем (день 1) — Базовый функционал, тестируем
  • 2. Докручиваем (день 2-7) — Обратная связь, уроки, кейсы
  • 3. Масштабируем лучшее — Копируем успешные паттерны
Пример: День 1 → получаем урок → День 2 → применяем урок → День 3 → используем кейс → результат лучше с первого раза!

Агент учится на каждом запросе. Чем больше используешь — тем лучше работает.

Система самообучения:

• При следующей задаче → использовать опыт
Безопасность:
ЗАПРЕЩЕНО: Удалять файлы без подтверждения
ВСЕГДА: Предлагать план, спрашивать подтверждение

МАГИЯ САМООБУЧЕНИЯ

Как агент учится на твоих запросах

  • Даём задачу → Агент выполняет
  • Запрашиваем оценку → Получаем обратную связь
  • Агент сохраняет урок в `_ЛОГИ/`
  • Следующая задача → Агент применяет урок
  • Отличный результат → Сохраняет кейс в `_КОНТЕКСТ/примеры/`
Пример по дням:

День 1: Задача → Агент пишет → "Добавь цифры" → Урок сохранён
День 2: Задача → Агент проверяет логи → Пишет с цифрами → "Отлично!" → Кейс сохранён
День 3: Задача → Агент использует кейс → Результат лучше с первого раза!

Это и есть система самообучения. Агент учится на твоих запросах и становится лучше с каждым использованием.

ROADMAP

ДЕЛАЕМ → ДОКРУЧИВАЕМ → МАСШТАБИРУЕМ

Процесс улучшения агента

ЭТАП 1: СНАЧАЛА ДЕЛАЕМ
Создаёшь базового агента: минимальный функционал, тестируешь на 2-3 задачах
ЭТАП 2: ДОКРУЧИВАЕМ
Собираешь обратную связь после КАЖДОЙ задачи. Агент сохраняет уроки в `_ЛОГИ/` и успешные кейсы в `_КОНТЕКСТ/примеры/`. За неделю: 10-20 задач, 5-10 уроков, 3-5 кейсов.
ЭТАП 3: МАСШТАБИРУЕМ ЛУЧШЕЕ
При создании нового агента копируешь успешные паттерны. Новый агент сразу умнее.
Пример: Создал Копирайтера → накопил 20 кейсов успешных постов → Создаёшь SMM-менеджера → копируешь лучшие кейсы → SMM-менеджер сразу пишет как Копирайтер после 20 итераций

Каждый новый агент начинает с уровня, до которого дорос предыдущий.

ТОП-5 ОШИБОК ПРИ СОЗДАНИИ

Что не делать

Ошибки структуры:
• Нет папки агента (`агенты/[АГЕНТ]/`)
• Неправильное имя файла (должно быть `.mdc`)
• Нет папок `_КОНТЕКСТ/примеры/` и `_ЛОГИ/`
Ошибки в правилах:
• Слишком длинные инструкции (агент теряется)
• Нет секции "НЕ входит" (агент берёт лишнее)
• Нет системы самообучения (агент не учится)
Ошибки использования:
• Нет обратной связи после задач
• Игнорирование логов
• Ожидание идеального результата с первого раза
Ссылка: ЧАСТЫЕ_ОШИБКИ_И_РЕШЕНИЯ.md

Избегай этих ошибок — агент будет работать лучше с первого дня.
